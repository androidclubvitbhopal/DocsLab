# Welcome to DocsLab!

Whether you are here to learn a new skill or contribute to an open-source project, you are in the right place. DocsLab is designed to be beginner-friendly, structured, and easy to navigate.

## 🧭 Choose Your Path

Not sure where to begin? Select what you want to do:

- **[I want to contribute](/docs/contributor-guide)**: Start here if you want to write documentation or code for DocsLab.
- **[I want to learn Open Source](/docs/open-source)**: Start here if you want to learn Git, GitHub, and open-source workflows.
- **[I want Android resources](/docs/android)**: Start here for mobile development roadmaps and tutorials.
- **[I want Web Development resources](/docs/web-dev)**: Start here for frontend, backend, and React/Next.js guides.
- **[I want Prompt Engineering resources](/docs/prompt-engineering)**: Start here to learn how to effectively talk to AI models.
- **[I want Productivity Shortcuts](/docs/shortcuts)**: Start here for IDE cheat sheets.
- **[I want to see Projects](/docs/projects)**: Start here to check out what the Android Club is building.

## 🛠 Making Your First Contribution

If you want to contribute but don't know where to start, we highly recommend reading our documentation guides. You don't need to know advanced coding to help out!

1. Check out the [Documentation Contribution Guide](/docs/contributor-guide/writing-documentation).
2. Read the [GitHub Contribution Guide](/docs/open-source/github-contribution-guide).
3. Find an open issue on our [GitHub repository](https://github.com/androidclubvitbhopal/DocsLab/issues).
4. Submit your first Pull Request!

<Callout type="info" title="Need Help?">
  Join our community discussions on GitHub or reach out to the core team if you
  get stuck. We are always happy to help new contributors.
</Callout>
